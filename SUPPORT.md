# Support

## Getting help

Before opening an issue:

1. Read the project README and the documentation index in `docs/README.md`.
2. Check the examples in `examples/` and the Streamlit application in `apps/`.
3. Search existing GitHub issues for the same error or request.
4. Confirm that the problem still occurs with a currently supported Python version and up-to-date dependencies.

## Bug reports

A useful bug report includes:

- a minimal reproducible example;
- the expected and actual result;
- the full error message or traceback;
- Python and operating-system versions;
- relevant package versions;
- sample data that is synthetic or safe to share.

## Questions and feature requests

Use GitHub Discussions when enabled for general questions, design ideas, or requests for guidance. Use GitHub Issues for reproducible defects and clearly scoped feature requests.

## Data and privacy

Do not upload confidential, personal, regulated, or proprietary datasets. Replace sensitive information with a small synthetic example that reproduces the behaviour.

## Commercial support

This is an open-source reference project and does not include guaranteed response times or production support commitments.
